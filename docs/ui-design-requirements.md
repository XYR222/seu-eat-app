# 东南今天吃点啥：当前前端完整设计说明

## 1. 产品概述

“东南今天吃点啥”是一个移动端优先的校园吃饭决策 Web App。当前版本面向黑客松演示，核心体验是帮助学生在校园食堂中快速决定吃什么。

当前产品由游客身份、本地个人数据、共享反馈数据和 AI 推荐能力共同组成：

- 游客身份用于区分本机不同演示用户。
- 个人数据包括 memory、收藏、最近吃过、不再推荐。
- 共享反馈包括菜品点赞、不推荐、标签、评论，以及窗口点赞、不推荐、评论。
- AI 帮选基于菜品数据、用户 memory、同学反馈和轻量 RAG 生成推荐。

当前主界面保持三 Tab：

- AI帮选
- 逛食堂
- 抽一餐

“我的”和“活动”作为顶部快捷入口，不占用底部 Tab。

## 2. 技术与页面形态

前端框架：

- Next.js App Router
- React
- TypeScript
- Tailwind CSS

产品形态：

- 单页移动端 Web App。
- 页面最大宽度为移动端卡片式布局。
- 底部固定三 Tab 导航。
- 详情、活动、我的均使用 bottom sheet。

主入口：

- `/login`：游客身份选择页。
- `/`：应用主界面。

## 3. 全局布局

主页面由 `AppShell` 控制。

页面结构：

1. 全局背景。
2. 左上角游客身份区。
3. 右上角快捷按钮区。
4. 当前 Tab 内容区域。
5. 底部 Tab 导航。
6. 根据状态打开的 bottom sheet。

左上角游客身份区包含：

- 当前游客名。
- 退出。

右上角快捷按钮区包含：

- 活动。
- 我的。

当前 Tab 内容区域顶部预留固定按钮安全空间，避免顶部按钮遮挡页面标题卡。

底部 Tab 固定包含：

| Tab key | 主文案 | 副文案 | 页面 |
| --- | --- | --- | --- |
| `ai` | AI帮选 | 问一句 | `AiRecommendTab` |
| `explore` | 逛食堂 | 自己找 | `ExploreTab` |
| `draw` | 抽一餐 | 不纠结 | `DrawMealTab` |

## 4. 登录页

登录页组件为 `LoginPanel`。

页面目标是让演示用户不使用真实邮箱，直接选择游客身份进入应用。

页面内容：

1. 产品名。
2. 登录说明。
3. 四个游客身份按钮。
4. 公共反馈同步说明。

游客账号：

- 游客 A
- 游客 B
- 游客 C
- 游客 D

游客按钮行为：

1. 点击游客按钮。
2. 前端请求 `/api/guest-session`。
3. 服务端写入游客 cookie。
4. 前端写入本地游客 id。
5. 跳转到首页。

按钮状态：

- 默认状态：显示游客名、描述和标签。
- Loading 状态：显示进入中。
- 请求失败：停留当前页。

## 5. AppShell 全局状态

`AppShell` 维护当前应用的主要前端状态。

核心状态：

| 状态 | 作用 |
| --- | --- |
| `tab` | 当前底部 Tab |
| `feedback` | 菜品反馈 |
| `stallFeedback` | 窗口反馈 |
| `memory` | 用户饮食记忆 |
| `history` | 我的最近记录 |
| `favorites` | 收藏菜品 |
| `deviceId` | 匿名设备 id |
| `myOpen` | 我的面板是否打开 |
| `activityOpen` | 活动面板是否打开 |
| `seenActivityIds` | 当前游客已读活动 |
| `exploreQuery` | 从活动跳转到逛食堂时带入的搜索词 |
| `selectedMyFoodId` | 从我的面板打开的菜品 |
| `selectedMyStallKey` | 从我的面板打开的窗口 |

页面加载后读取：

- memory
- 菜品反馈本地缓存
- 窗口反馈本地缓存
- 最近记录
- 收藏
- 活动已读状态
- 共享反馈 snapshot

本地数据按游客身份隔离。

## 6. 顶部活动入口

活动按钮显示在主界面右上角，与“我的”按钮同组。

按钮文案：

```text
活动
```

- 打开活动面板后，当前活动全部标记为已读。

2. 记录当前活动 id 为已读。
3. 红点消失。

## 7. 活动面板

活动面板组件为 `ActivitySheet`。

打开方式：

- 点击顶部“活动”按钮。

面板内容：

1. 顶部拖拽条。
2. 小标题：食堂动态。
3. 主标题：近期活动。
4. 说明：优惠、新菜、窗口提醒和高峰信息。
5. 活动卡片列表。

活动卡片字段：

| 字段 | 说明 |
| --- | --- |
| `type` | 活动类型 |
| `title` | 活动标题 |
| `canteen` | 食堂 |
| `stall` | 窗口，可为空 |
| `period` | 活动时间 |
| `summary` | 活动摘要 |
| `tags` | 活动标签 |

活动类型包括：

- 限时优惠
- 新菜上架
- 高峰提醒
- 窗口变动
- 活动套餐

活动卡片按钮：

```text
去看看
```

点击“去看看”：

1. 关闭活动面板。
2. 切换到底部 Tab：逛食堂。
3. 将活动对应的食堂/窗口写入搜索词。
4. 逛食堂页面展示对应搜索结果。

## 8. 我的入口

我的按钮显示在主界面右上角，与“活动”按钮同组。

按钮文案：

```text
我的
```

点击行为：

- 打开 `MyRecordsSheet`。

我的入口展示当前游客自己的本地记录，不展示其他游客的个人数据。

## 9. 我的面板

我的面板组件为 `MyRecordsSheet`。

打开方式：

- 点击顶部“我的”按钮。

面板内容：

1. 顶部拖拽条。
2. 标题区。
3. 三个统计卡片。
4. 最近吃过列表。
5. 收藏菜品列表。
6. Memory 摘要。
7. 最近吃过 memory chips。

统计卡片：

| 卡片 | 数据 |
| --- | --- |
| 最近记录 | `history.length` |
| 收藏菜品 | `favorites.length` |
| 不再推荐 | `memory.avoidFoods.length` |

最近吃过列表：

- 最多展示最近 5 条。
- 每条显示菜名、食堂、窗口、来源、动作、相对时间。
- 点击记录打开菜品详情。

收藏菜品列表：

- 显示当前游客收藏的菜品。
- 点击菜品主体打开菜品详情。
- 点击“取消收藏”移除收藏。

Memory 区域：

- 复用 `MemorySummary`。
- 可以删除单个 memory chip。
- 可以清空 memory。

## 10. AI帮选页面

AI帮选页面组件为 `AiRecommendTab`。

页面目标是让用户通过自然语言输入获得 3 个具体菜品推荐。

页面结构：

1. 顶部说明卡。
2. AI 输入卡。
3. Memory 摘要。
4. 当前条件提示。
5. 追问按钮区。
6. 推荐结果区。
7. 菜品详情 Sheet。
8. 窗口口碑 Sheet。

### 10.1 顶部说明卡

显示内容：

- 产品名。
- 页面标题：AI 帮你选饭。
- 状态标签。
- 页面说明。

### 10.2 AI 输入卡

输入控件：

- 多行 textarea。

默认输入示例：

```text
15元以内，清淡，不要太咸，离教学楼近
```

快捷条件：

- 15元以内
- 清淡
- 不辣
- 高蛋白
- 离我近

快捷条件点击行为：

- 将对应条件追加到当前输入文本。

主按钮：

```text
生成 3 个选择
```

点击行为：

1. 设置 loading。
2. 请求 `/api/recommend`。
3. 传入 query、memory、feedback、stallFeedback、session。
4. 接收 recommendations、resolvedIntent、memoryPatch。
5. 渲染推荐卡。
6. 合并 memoryPatch。
7. 结束 loading。

Loading 文案：

```text
正在检索菜单和同学反馈...
```

### 10.3 追问按钮

追问按钮包括：

- 换一批
- 再便宜点
- 不要面食
- 离我更近
- 别太辣
- 偏咸

点击行为：

- 使用当前输入文本加追问条件重新请求推荐。

### 10.4 推荐结果

推荐结果使用 `RecommendationCard`。

每张卡片显示：

- 推荐序号。
- 距离标签。
- 价格。
- 菜名。
- 食堂 / 窗口。
- 匹配分。
- 菜品标签。
- 推荐理由。
- 可能缺点。
- 推荐证据。
- 主按钮：就吃这个。
- 次按钮：不喜欢。
- 快速反馈 chip。

推荐卡点击行为：

- 点击卡片主体：打开菜品详情。
- 点击“就吃这个”：写入 memory.recentFoods，并写入历史。
- 点击“不喜欢”：写入 avoidFoods。
- 点击快速反馈 chip：更新 memory 偏好或避开标签。

## 11. Memory 摘要组件

Memory 摘要组件为 `MemorySummary`。

显示来源：

- AI帮选页面。
- 我的面板。

显示字段：

- `budgetMax`
- `preferTags`
- `avoidTags`
- `recentFoods`

展示形式：

- 每条 memory 以 chip 显示。
- 可删除的 chip 带删除符号。
- 预算 chip 仅展示，不作为可删除项。

按钮：

```text
清空
```

点击行为：

- 清空当前游客 memory。

## 12. 逛食堂页面

逛食堂页面组件为 `ExploreTab`。

页面目标是让用户手动搜索或按食堂层级浏览菜品。

页面结构：

1. 顶部说明卡。
2. 搜索框。
3. 标签筛选区。
4. 食堂/窗口/菜品层级浏览区。
5. 热门 Top 5。
6. 菜品列表。
7. 菜品详情 Sheet。
8. 窗口口碑 Sheet。

### 12.1 搜索

搜索框支持匹配：

- 菜名。
- 食堂。
- 窗口。
- 描述。
- 标签。

搜索框 placeholder：

```text
搜索菜品 / 食堂 / 窗口 / 标签
```

输入行为：

- 输入时实时筛选菜品列表。

### 12.2 标签筛选

筛选标签：

- 全部
- 清淡
- 不辣
- 高蛋白
- 热汤
- 面食
- 米饭
- 性价比高

点击行为：

- 点击某个标签后，只显示包含该标签的菜品。
- 点击“全部”清除标签筛选。

### 12.3 食堂层级浏览

初始展示食堂卡片。

食堂卡片显示：

- 食堂序号。
- 食堂名。
- 菜品数量。
- 窗口数量。

点击食堂卡片：

1. 设置 `selectedCanteen`。
2. 展示该食堂下的窗口列表。

### 12.4 窗口层级浏览

窗口卡片显示：

- 窗口名。
- 所属食堂。
- 菜品数量。

点击窗口卡片：

1. 设置 `selectedBrowseStall`。
2. 展示该窗口下的菜品列表。

窗口菜品列表顶部按钮：

- 返回窗口列表。
- 查看窗口口碑。

点击“返回窗口列表”：

- 清空当前窗口选择。

点击“查看窗口口碑”：

- 打开 `StallDetailSheet`。

### 12.5 热门 Top 5

热门菜品根据点赞和不推荐排序。

热门卡片显示：

- 排名。
- 菜名。
- 价格。
- 食堂。
- 点赞数。
- 不推荐数。

点击热门卡片：

- 打开菜品详情。

### 12.6 菜品列表

菜品卡片显示：

- 菜名。
- 食堂 / 窗口。
- 价格。
- 点赞数。
- 不推荐数。
- 前 3 个标签。

点击菜品卡片：

- 打开菜品详情。

## 13. 抽一餐页面

抽一餐页面组件为 `DrawMealTab`。

页面目标是给用户一次生成三个随机但受 memory 约束的吃饭选择。

页面结构：

1. 顶部说明卡。
2. 抽卡控制卡。
3. 三张抽卡结果。
4. 菜品详情 Sheet。
5. 窗口口碑 Sheet。

### 13.1 抽卡控制

控制卡显示：

- 当前抽卡已避开的内容。
- 主按钮。

已避开内容来自：

- `memory.avoidTags`
- `memory.recentFoods`

主按钮初始文案：

```text
开始抽卡
```

抽过后文案：

```text
再抽一轮
```

点击行为：

1. 调用 `drawMealCards`。
2. 生成三张卡。
3. `drawCount + 1`。
4. 可以重复抽取。

### 13.2 三张抽卡结果

三张卡类型固定：

| type | 展示名 | 含义 |
| --- | --- | --- |
| `safe` | 稳妥卡 | 稳定、低风险 |
| `explore` | 探店卡 | 换窗口、尝试新选择 |
| `surprise` | 惊喜卡 | 随机感更强 |

每张卡显示：

- 卡片标题。
- 类型标签。
- 菜名。
- 食堂 / 窗口。
- 价格。
- 标签。
- 抽中理由。
- 按钮：就它了。

点击行为：

- 点击卡片主体：打开菜品详情。
- 点击“就它了”：写入 recentFoods，并写入历史。

## 16. 共享反馈交互

当前共享反馈通过前端动作触发。

菜品层级反馈：

- 点赞。
- 不推荐。
- 标签反馈。
- 评论。

窗口层级反馈：

- 点赞窗口。
- 不推荐窗口。
- 窗口评论。

用户点击反馈后：

1. 立即更新 React state。
2. 写入本地缓存。
3. 请求共享反馈 API。
4. API 成功后刷新共享反馈 snapshot。
5. API 失败时保留本地状态。

## 19. 当前主要用户路径

### 路径 A：游客进入应用

1. 打开 `/login`。
2. 点击任意游客身份。
3. 进入首页。
4. 默认显示 AI帮选。

### 路径 B：AI 帮选吃什么

1. 打开 AI帮选。
2. 输入自然语言需求。
3. 点击生成 3 个选择。
4. 查看 3 张推荐卡。
5. 点击某张卡打开菜品详情。
6. 点击就吃这个写入记录。

### 路径 C：手动逛食堂

1. 打开逛食堂。
2. 搜索菜品或点击食堂卡片。
3. 进入窗口列表。
4. 进入窗口菜品列表。
5. 打开菜品详情或窗口口碑。

### 路径 D：抽一餐

1. 打开抽一餐。
2. 点击开始抽卡。
3. 查看稳妥卡、探店卡、惊喜卡。
4. 点击卡片看详情。
5. 点击就它了写入记录。

### 路径 E：查看个人记录

1. 点击我的。
2. 查看最近吃过、收藏、memory。
3. 点击历史或收藏菜品打开详情。
4. 删除 memory chip 或清空 memory。

### 路径 F：查看食堂活动

1. 看到活动按钮红点。
2. 点击活动。
3. 查看近期活动。
4. 点击去看看。
5. 自动跳到逛食堂并带入对应搜索词。
